# Tradvance

> 해외 바이어 문의를 붙여넣으면 확인 항목을 분류하고, 위험 구간을 짚어주는 영어 응대 초안을 생성하는 도구

🔗 **배포 링크**: https://tradvance-korea.vercel.app/
🛠 **기술 스택**: Next.js (App Router) · TypeScript · Tailwind CSS · Supabase

핵심 원칙: **AI(자동화 로직)는 초안 생성까지만, 확인·발송은 담당자가 직접 한다.**

---

## 1. 왜 만들었나

- 수출 담당자가 부족한 소규모 **제조업체**(산업용 부품·기계·원자재, 직원 10~50인)를 위한 문의 응대 보조 도구
- 기획 범위를 2단계로 축소
  1. 수출 전체 지원 → **문의 수신 이후 응대 준비 과정**으로 축소
  2. 전 업종 대상 → **제조업**으로 축소 (바이어 발굴·콜드메일은 Rinda 등 기존 서비스 영역과 겹쳐 제외)

### 문제 정의
- 소규모 제조업체는 해외 바이어 문의 시 확인할 항목이 많음 (견적, MOQ, 납기, 인증, 물류 조건 등)
- 확인 항목이 많아 답변이 지연되거나, 확정되지 않은 정보를 확정된 것처럼 전달할 위험 존재
- 답변 지연·오안내 → 바이어 신뢰 하락, 거래 기회 손실로 이어질 수 있음
- **핵심 문제**: 해외 바이어 문의 응대는 단순 번역이 아니라 `확인 항목 정리 + 영어 답변 작성 + 오안내 리스크 관리`가 함께 필요한 업무

## 2. 근거

### 사용자 조사 (9명, 2026.7.3)
| 확인한 내용 | 결과 |
|---|---|
| 사람 개입 구조 선호 | 9명 중 8명 (89%) |
| AI 답변 과신 우려 (최다 우려) | 9명 중 6명 (67%) |
| 가장 많이 언급된 확인 항목 | 가격/견적 |
| 주요 어려움 | 내부 확인 반복, 견적 조건 확정, 영어 표현 부담 |

- 인사이트: 사용자는 초안 자체에는 관심이 있으나 **자동 발송에는 불안감**을 느낌 → 담당자 검토형 구조로 설계
- 보조 통계: 전경련중소기업협력센터(2018, 1,020개사) — 응답 기업 34.6% 바이어 발굴 애로 응답 (발굴 단계 애로로, 본 서비스의 응대 단계와는 인접 참고 자료)

### 경쟁 환경
| 구분 | 주요 서비스 | Tradvance와의 차이 |
|---|---|---|
| 바이어 발굴 | Rinda, EC21, 고비즈코리아 | 발굴 단계 집중, 제조·화장품·식품 등 범용 타겟 |
| AI 세일즈 자동화(해외) | Apollo.io, Outreach.io, Instantly.ai 등 | 콜드메일·아웃바운드 중심, 산업재 응대 특화 사례 없음 |
| **Tradvance** | 해외 바이어 문의 응대 보조 | 제조업 문의 수신 후 응대 준비 단계에 특화 |

## 3. 해결 방안

### 사용 흐름
1. 바이어 문의 원문 붙여넣기 (로그인·연동 불필요)
2. 7개 확인 항목 자동 분류 + 영어 답변 초안 생성
3. 위험 구간(가격·납기·사양 등 확정되지 않은 정보) 하이라이트
4. 한국어 자동 번역 확인 — 한국어 수정 시 영문 초안에 자동 반영
5. 위험 구간 전부 확인 후 초안 복사 (발송 기능 없음)

### 확인 항목 7개
`가격/견적` `MOQ` `납기/리드타임` `기술 사양` `포장/물류` `인증/품질 기준` `결제 조건`

### 유저플로우 요약
- 진입점 1개(랜딩 → 입력), 로그인·회원가입 없음
- 이탈 가능 지점 2곳: ① 문의 미인식 시 재입력 유도 ② 위험 구간 미확인 시 복사 불가
- 서비스가 관여하는 마지막 지점은 "복사 완료"까지 — 실제 발송은 서비스 밖의 행동

## 4. 화면 구성

### 랜딩 페이지 (`/`)
- 헤더: 로고, "사용해보기" CTA → `/tool` (페이드 전환)
- 히어로: 배경 슬라이드쇼(Ken Burns 효과), 타이핑 헤드라인 3종 순환
- 서비스 소개: 답변 초안 미리보기 카드(위험 구간 하이라이트 예시)
- 한→영 자동 반영 데모: "영어가 부담스럽다구요? 괜찮습니다!" 순환 애니메이션
- 핵심 기능 3카드: 분류 · 위험 구간 체크리스트 · 초안 작업(발송 없음)
- 동작 순서 3단계 카드 + 화살표 커넥터
- 신뢰 지표: 89% / 67% / 7개 카테고리 (스크롤 진입 시 카운트업, 재진입 시 재실행)
- 푸터: 개인정보 안내 문구("입력한 문의는 분석 용도로만 사용, 서버 미저장")

### 도구 페이지 (`/tool`)
| 화면 | 기능 |
|---|---|
| 입력 화면 | 문의 원문 붙여넣기, 샘플 3종(MOQ/납기/인증) 자동 채움, "분석 시작하기" |
| 로딩 화면 | 스피너 + 안내 문구 (최소 0.5초 노출) |
| 예외 화면 | 문의 미인식 시 재입력 유도 |
| 결과 화면 | 4단계 아코디언: ①원문+한글 번역 ②7개 카테고리 분류 ③영문 초안(위험 구간 하이라이트)+한글 편집·재반영 ④위험 구간 전체 확인 후에만 복사 가능 |
| 위험 구간 모달 | 항목별 근거 설명 + 체크리스트, 전체 체크 시에만 "확인 완료" 활성화 |
| 하단 액션 | 처음으로 / 메인으로 / 초안 복사하기 + 복사 완료 토스트 |

### 반응형 지원
| 구간 | 대응 내용 |
|---|---|
| 모바일(375px) | 가로 스크롤 없음, 지정 위치 줄바꿈, 엄지 터치 고려한 버튼 배치 |
| 태블릿(768px) | 카드 높이·문구 위치 정렬, 카테고리 배지 줄바꿈 여유 확보 |
| 데스크톱(1280px+) | 기존 레이아웃 100% 유지 (반응형 작업 시 픽셀 단위로 검증) |

## 5. 공통 백엔드 로직

| 기능 | 설명 |
|---|---|
| 문의 인식 판별 | 최소 길이 + 카테고리 키워드/신호어/물음표 존재 여부로 판정 |
| 카테고리 분류 | 7개 카테고리 키워드 매칭 |
| 초안 생성 | 카테고리별 고정 템플릿 문장 조합 + 원문 추출 수치 반영 |
| 위험 구간 탐지 | 초안 내 미확정 표현(가격/기간/수량)을 정규식으로 탐지 |
| 위험 사유 설명 | 카테고리별 사전 정의 근거 문구 (추후 LLM 연동 대비 함수 분리) |
| 번역 | 무료 Google 번역 API, 고정 인사말/맺음말은 자연스러운 한국어로 직접 대체 |
| 참여 통계/이벤트 로깅 | 복사 완료 시 카테고리·위험 건수 기록, 주요 행동 Supabase 로깅 |

> ⚠️ **구현 상태 참고**: 이번 프로토타입은 카테고리 분류·초안 생성·위험 구간 탐지·이유 설명을 전부 **규칙/템플릿 기반**으로 구현했다. LLM(Claude API 등)은 연동되어 있지 않으며, `explainRisk` 등 일부 로직은 향후 LLM 연동을 고려해 함수를 분리해둔 상태다. 정확성과 데모 안정성을 우선한 의도적 선택이다.

## 6. MVP 범위

### 포함
- 바이어 문의 원문 입력
- 확인 항목 자동 분류 (7개 카테고리)
- 영어 답변 초안 생성 + 한국어 자동 번역/수정 반영
- 위험 구간 체크리스트
- 담당자 확인 후 복사

### 제외 (및 이유)
| 제외 항목 | 이유 |
|---|---|
| AI 자동 발송 | 설문에서 자동 발송 불안감 확인, 확정 필요 정보를 AI가 바로 보내는 건 위험 판단 → 복사까지만 지원 |
| Gmail OAuth 실연동 | 이번 검증 핵심은 "메일 연동 편의성"이 아니라 "문의 분석·위험 탐지 유용성" → 포워딩 가이드로 대체 |
| 바이어 발굴 | Rinda/EC21/고비즈코리아 등과 영역 중복, 개인 미니 프로젝트로 검증하기엔 DB·영업 자동화 범위가 큼 |
| 견적/납기 자동 확정 | 회사 내부 기준·재고·생산 일정에 따라 달라져 AI 자동 확정은 위험 → 위험 구간 표시로 대체 |
| 계약 클로징 지원 | 설문에서 "계약 단계는 사람 대 사람 신뢰가 중요"하다는 의견 확인 → 1차 문의 응대 범위를 넘어서 장기 로드맵으로 이관 |
| 대시보드/문의 보드 | 단일 흐름으로 축소하며 제외 |
| 제조업 외 업종(화장품·식품 등) | 확인 항목 표준화가 어려워 이번 범위에서 제외, 산업재로 좁혀 차별화 |

## 8. 제작 회고

**잘된 점**
- 넓은 수출 지원 서비스 → "문의 응대" → "제조업" 순으로 두 단계에 걸쳐 범위를 좁혀 검증 가능한 문제로 정리
- 영어 표현 부담을 덜기 위한 한국어 수정 기반 초안 갱신 기능까지 구현

**아쉬운 점**
- 설문 표본 9명으로 크지 않고, 실제 제조업 수출 담당자만을 대상으로 한 검증은 부족
- 실사용 데이터 부재로 답변 시간 단축 효과는 정량 검증하지 못함
- 규칙/템플릿 기반 구현이라 실제 LLM 연동 시의 유연성은 검증하지 못함
- 처음부터 웹 중심 기획으로 모바일 최적화는 후순위로 진행

**다음에는 이렇게 할 것**
- 실제 제조업 수출 담당자 인터뷰 확대
- 위험 구간 탐지 정확도 정량 검증
- 초안 수정률 기반 유용성 측정
- 핵심 로직(위험 사유 설명 등) LLM 연동 검토
